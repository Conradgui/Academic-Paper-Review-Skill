## 已验证行为

- 干净稿只输出轻量文本质量结论，不强行生成 `T-*` finding。
- 严重稿同时输出科学性 finding、详细文本 finding、严重度、确定性、主张校准和修改行动表。
- Delta Review 使用原 finding ID、精确状态枚举、强制状态矩阵，并把未关闭项带入行动表。
- 全文润色不覆盖原稿，生成新稿和伴随报告；保留样本量、系数、p 值、变量、公式、引用与非因果边界。
- 未提供作者样本时，报告明确使用中性或论文一致性基线，不宣称匹配作者个人文风。
- `paper-reviews/` 和 `paper-revisions/` 未混入内部计划或日志。

## 评测中发现并修复的问题

1. 最初的“干净”理论夹具实际含有未形式化的局部/全局概念、未使用的有限性条件和充分/必要条件混淆。Skill 正确识别后，夹具被修正，没有降低审阅标准。
2. 孤立术语问题会被扩写为完整 `T-01`。协议现明确轻量观察不分配 T-series ID。
3. Delta Review 曾使用中文自由状态和 `D-01`。协议现强制精确状态 token、原 ID 和状态矩阵。
4. 内部 `plan.md` 曾进入 `paper-reviews/`。输出目录现只允许用户交付物，evaluator 单独验证目录卫生。
5. 部分早期会话读取了全局旧版 Skill。runner 现使用临时 `CODEX_HOME` 和 Skill 哈希验证，彻底隔离全局安装。

## 证据边界

- 这是四个合成任务各一次独立运行，不构成统计显著的模型稳定性 Benchmark。
- 结果证明当前模型与当前 Skill 在四条核心路径上可用，但不能外推到所有领域、模型版本和长篇复杂论文。
- 原始 JSONL、stderr 和合成输出保存在本地 `_project-records/forward-evals/`，因体积与运行噪声被 Git 忽略。
