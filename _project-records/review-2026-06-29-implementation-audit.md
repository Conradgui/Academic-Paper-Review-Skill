# Paper Review Skill 实现复核记录

日期：2026-06-29

## 复核范围

- 通用版 `$paper-review` 的双线审阅、文本质量路由、输出契约、外部检索、作者化润色和格式边界。
- `proofing_scan.py` 的输入、候选规则、错误处理和降噪行为。
- README 与 Skill 实际行为的一致性。
- `$latex-paper-review` 专项目录隔离。

## 已发现并修正

1. 外部检索存在绕过 opt-in 的歧义；现改为未获用户明确授权不得联网，无法判断时先标记 `Needs External Verification`。
2. 中英文专项 reference 原本几乎会对所有稿件加载；现改为默认轻量检查，详细审阅触发后才加载语言专项规则。
3. 详细文本质量表缺少全局契约要求的严重程度与判断确定性；现已统一。
4. “作者文风”可能被稿件内部风格误推断；现区分已确认作者文风、论文一致性和中性学术三种基线。
5. DOCX 全文润色承诺过强；现要求具备编辑和渲染核验工具，否则降级为结构化修改文本或其他可编辑源。
6. Proofing 会静默吞掉损坏 DOCX 和不可提取文本的 PDF；现改为明确失败或警告。
7. 语义候选重复告警较多；现仅合并邻近的过强主张和聊天残留，确定性标点错误仍逐条报告。
8. 常见论文写作问题覆盖不足；现补充章节级失败模式、中英文表达规范和实证统计报告风险。

## 常见论文问题覆盖

- 摘要、正文、关键词和结论不一致。
- 引言没有形成清晰研究缺口、问题和贡献边界。
- 文献综述逐篇罗列，缺少比较、分类、争议和研究缺口综合。
- 方法缺少数据或材料来源、纳排规则、预处理和可复核细节。
- 结果只强调显著性，忽略效应量、不确定性、空结果或边界结果。
- 讨论把解释当作事实，缺少替代解释、局限和外部有效性边界。
- 结论引入新证据、新机制或强于结果的主张。
- 中文指代、平行结构、全半角、数字与单位、缩写首次定义和空泛动词问题。
- 英文主谓一致、冠词与可数性、指代、平行结构、搭配、英美拼写和缩写首次定义问题。
- 伦理审批、知情同意、利益冲突、资金与数据/代码声明在适用场景下缺失。

## 剩余风险

- 前向评测目前是四个合成任务各一次运行，尚未覆盖跨模型、重复运行方差、超长论文和更多学科类型。
- DOCX 测试覆盖最小有效文件和损坏主文档，尚未覆盖带批注、修订、域代码、复杂表格和公式的真实文档。
- 全文润色的语义保真依赖账本和复核协议，尚无确定性工具自动证明所有事实、数字、引用和主张边界均未改变。
- PDF 文本提取依赖 `pypdf`；扫描版需要 OCR，复杂版面仍可能丢失阅读顺序。

## 结论

当前实现达到可发布的本地质量门槛，没有发现阻塞项。对外应表述为“结构化预审与受保护润色工作流”，不应承诺发现所有论文错误、还原未经确认的作者个人文风，或保证复杂文档完全保真。
